Description

Examples of motility impairments

  • Paraplegia - Loss of control of the legs
  • Quadrplegia - Loss of control of arms and legs
  • Cerebral palsy - Symptoms include "spasms, involuntary movement, impaired speech, and even paralysis" (Polacek, Sporka, & Slavik, 2017)
  • Muscular dystrophy - progressive degeneration of muscles which can lead to paralysis (Polacek, Sporka, & Slavik, 2017)
  • Multiple sclerosis - can experience tremors, spasticity, or muscle stiffness (Polacek, Sporka, & Slavik, 2017)
  • Arthritis - Pain in joints. while most common in the knees this condition can also exists in arms and the hands (Polacek, Sporka, & Slavik, 2017)
  • Parkinson's Disease - cause uncontrollable tremors and affect the voice in more severe cases as well (Polacek, Sporka, & Slavik, 2017)
  • Amputees - a loss of an arm makes it difficult to use a keyboard and a mouse at the same time.

Challenges

Types of computer input

  • Text Input The primary text input is keyboards on PCs and touchpads for smartdevices
  • Selection The primary selection mechanism is the mouse, trackpads. The ability to move a mouse around a graphic interface

Types of challenges

  • Able to use main input forms, but not quickly or accurately enough (double clicking, or holding shift when captalizing a word)
  • Unable to make large movements with one's arm. Makes mouse difficult to use
  • Unable to interact with a keyboard at all
  • Unable to speak- makes speech recognition software ineffective

Software Accomodations

  • Voice Control is rapidly improving

Keyboard Modifications

  • Utilities that hold down the commands such as Shift or Control
  • A Key Repeat Function- Helps those who "cannot release keys quick enough to avoid multiple selections" https://www.washington.edu/doit/how-can-people-mobility-impairments-operate-computers
  • Abbreviation Software
  • Text prediction systems

Hardware Accomodations

Keyboard and Mouse alternatives

  • Ambiguous keyboards- keyboards where multiple characters are on single key
  • Mini Keyboards
  • Trackballs
  • Foot Mice
  • Moore Code input
  • Eye Tracking devices- A device that shows promise but is usually best when paired with another movement
  • Brain Computer Interfaces

Developer Tips

  • "Changing the setting of any user interface component does not automatically cause a change of context unless the user has been advised of the behavior before using the component." (WCAG requirement 3.2.2)
  • Allow for keyboard alternatives to mouse use when possible and reducing the number of keystrokes required to navigate the site
